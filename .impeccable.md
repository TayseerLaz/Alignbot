# Hader / ALIGNED — Design Context

> Source of truth for the UX/UI of the platform. All design work follows this.
> Established 2026-06-22 with the product owner. Direction: **neutral-minimal,
> Linear-class, compact.**

## Design Context

### Users
A multi-tenant WhatsApp/Messenger/Instagram/voice commerce + AI-bot platform.
Distinct audiences, each with a different job and tempo:
- **Operators (Editor/Viewer)** — live in the **inbox** all day, replying across
  channels, toggling the AI, handing off. Often on a **phone**. Need speed,
  density, keyboard control. This is the highest-frequency surface.
- **Tenant admins/editors** — manage catalog (products/services/business info),
  build & deploy the bot, run broadcasts/contacts, wire integrations. Scanning
  and editing structured data.
- **Tenant viewers** — read-only oversight (analytics, audit, orders).
- **ALIGNED HQ super-admins** — operate **across all tenants**: suspend/control
  orgs, feature flags, provenance, revenue, system health. Power-dense.
- Market skews MENA/Gulf (Arabic + RTL aware) and international.

### Brand Personality
**Precise · quiet · fast.** The product should feel like an instrument a
professional reaches for, not a dashboard they tolerate. Confidence through
restraint: the UI gets out of the way and reports its own state at a glance.

### Aesthetic Direction
**Neutral-minimal, Linear/Stripe-class.** Near-monochrome neutral canvas; the
warm Oxblood brand is demoted from "dominant surface color" to a **rare, decisive
accent**. Compact density (operators + big-data admins scan a lot per screen).
- **Theme:** light default (warm-neutral, not stark white); dark fully supported,
  user-toggleable.
- **Color:** neutral grays (subtly oxblood-tinted, OKLCH) for ~90% of pixels;
  **Oxblood `#360516`** = primary action / active state; **Signal red `#e53e34`**
  = escalation/alert/destructive only. Accents work because they're rare.
- **Type:** Plus Jakarta Sans (UI/body) + **JetBrains Mono** (SKUs, prices, IDs,
  quantities, timestamps — tabular, so data feels precise). No display serif.
- **Shape:** tight radii (6–10px, not 14–28px), hairline 1px borders over heavy
  shadows, restrained elevation.
- **Motion:** fast and subtle — transform/opacity only, ease-out, 120–200ms;
  one well-orchestrated moment beats scattered micro-animations.
- **Anti-references:** the current soft/rounded/spacious look; generic AI tells
  (gradient text, colored left-border stripes, card-in-card, glassmorphism,
  cyan-on-dark, everything-a-card).

### Design Principles
1. **Density over whitespace.** Every screen earns its space. Compact rows,
   tabular numerics, no decorative empty gutters. "Remove the big spaces."
2. **Neutral canvas, rare accent.** Oxblood/Signal appear only for the single
   most important thing on a screen. If everything is emphasized, nothing is.
3. **Progressive disclosure.** Start simple, reveal power through interaction.
   The inbox and bot builder lead with the 20% people use, hide the rest a click away.
4. **The system reports its own state.** A persistent status strip
   (Bot · Channel · today's orders), real skeletons that match layout, optimistic
   updates, at-a-glance health — never make the user hunt to know if it's working.
5. **Keyboard-first for power users.** ⌘K command palette everywhere; arrow-key
   navigation in the inbox. Operators should rarely touch the mouse.
6. **One opinionated golden path.** New tenants get guided to a first win
   (Catalog → Bot → Connect channel) in <10 minutes, not a blank canvas.

### Hard constraints
- Next.js 15 (App Router) · React 19 · Tailwind v4 · basePath `/app`.
- Live multi-tenant production app — changes must be incremental & non-breaking,
  verified by `pnpm --filter @aligned/web build`.
- Re-theme via the existing semantic Tailwind tokens (stable names) so component
  churn stays low. RTL/Arabic-aware. WCAG AA contrast.
